# [1주차] 자바는 어떻게 실행될까? (그림으로 이해하는 JVM)

## 1. 자바 삼총사: JDK, JRE, JVM

JDK, JRE, JVM은 서로 별개의 도구가 아니라, 포함 관계로 이해하면 쉽습니다.

- **JVM**: 바이트코드를 한 줄씩 읽고 실행하는 엔진입니다. 운영체제마다 다르게 구현되어 있어서, 동일한 `.class` 파일이 윈도우와 맥에서 모두 동작할 수 있게 해줍니다.
- **JRE**: JVM과 자바 표준 라이브러리(`java.lang`, `java.util` 등)를 포함합니다. 자바 프로그램을 실행만 하려면 JRE만 있어도 충분합니다.
- **JDK**: JRE에 컴파일러(`javac`) 등 개발 도구를 더한 것입니다. 자바 코드를 직접 작성하고 컴파일하려면 JDK가 필요합니다.

```
JDK (자바 개발 키트)
 ├─ 개발 도구 (javac, javadoc, jar 등)
 └─ JRE (자바 실행 환경)
     ├─ JVM (바이트코드 실행 엔진)
     └─ 핵심 라이브러리 (java.lang, java.util 등)
```

> 정리하면, "실행만 할 것인가, 개발까지 할 것인가"로 구분하면 됩니다. 개발자는 보통 JDK를 설치하지만, 그 안에는 이미 JRE와 JVM이 모두 포함되어 있습니다.

## 2. `.java`가 실행되기까지의 흐름

자바의 "한 번 작성하면 어디서든 실행된다(Write Once, Run Anywhere)"는 철학은 **바이트코드**라는 중간 단계 덕분에 가능합니다.

```
.java 파일        →     javac 컴파일러     →     .class 파일      →     JVM
(소스 코드)              (소스 코드 변환)         (바이트코드)            (바이트코드 실행)
```

1. `.java` 파일은 사람이 읽는 텍스트일 뿐이며, 컴퓨터가 바로 이해할 수 없습니다.
2. `javac` 컴파일러가 이를 `.class` 파일(바이트코드)로 변환합니다. 이 바이트코드는 특정 OS의 기계어가 아니라, JVM이라는 가상의 컴퓨터가 이해하는 언어입니다.
3. JVM이 이 바이트코드를 읽어, 실제 OS와 하드웨어에 맞게 해석하고 실행합니다.

> 윈도우에서 컴파일한 `.class` 파일을 맥으로 그대로 옮겨 실행할 수 있는 이유가 바로 여기에 있습니다. JVM만 설치되어 있다면 동일하게 동작합니다.

## 3. 클래스 로더 맛보기

JVM은 `.class` 파일을 곧바로 실행하지 않습니다. **클래스 로더(Class Loader)**가 먼저 해당 파일을 메모리에 올려주는 과정을 거칩니다. 이 과정은 크게 3단계로 나뉩니다.

```
Loading  →  Linking  →  Initialization
```

| 단계                        | 설명                                                                                  |
| --------------------------- | ------------------------------------------------------------------------------------- |
| **Loading (로딩)**          | `.class` 파일을 디스크에서 찾아 읽어, JVM 메모리(메서드 영역)에 올립니다.             |
| **Linking (링크)**          | 읽어온 클래스가 문법적으로 올바른지 검증하고, 변수가 들어갈 메모리 공간을 준비합니다. |
| **Initialization (초기화)** | `static` 변수에 실제 값을 대입하고, `static` 초기화 블록을 실행합니다.                |

> **비유**: 이사하는 과정과 비슷합니다.
>
> - Loading = 짐을 트럭에서 내려 집 안으로 들여놓는 과정
> - Linking = 가구가 설계도대로 맞는지 확인하고 놓을 자리를 비워두는 과정
> - Initialization = 실제로 가구를 배치하고 전원을 연결하는 과정
